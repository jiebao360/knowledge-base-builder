## 📦 安装

### 方式 1: 从 Clawhub 安装（推荐）
```bash
安装 https://clawhub.ai/你的用户名/knowledge-base-builder
```

### 方式 2: 从 GitHub 安装
```bash
npx skills add 你的用户名/knowledge-base-builder -g -y
```

### 方式 3: 手动安装
```bash
# 克隆仓库
git clone https://github.com/你的用户名/knowledge-base-builder.git

# 复制到技能目录
cp -r knowledge-base-builder ~/.agents/skills/

# 验证安装
npx skills list
```

## ⚙️ 配置

### 基础配置
在 `config.json` 中配置：
```json
{
  "storage": "feishu",
  "space_name": "OpenClaw 个人知识库",
  "categories": 8,
  "auto_sync": true,
  "sync_time": "22:00"
}
```

### Content Collector 配置
```json
{
  "enabled": true,
  "mapping": {
    "技术教程": ["教程", "指南"],
    "实战案例": ["案例", "项目"],
    "热点资讯": ["资讯", "新闻"]
  }
}
```

## ⚠️ 注意事项

### 飞书权限
确保已授权以下权限：
- `bitable:app` - 多维表格
- `docs:doc` - 云文档
- `drive:file` - 云盘文件
- `wiki:space` - 知识库

### 文件大小
- 单个文档 ≤ 10MB
- 知识库总空间 ≤ 1GB
- 图片自动压缩

### 同步频率
- 手动同步：随时
- 自动同步：每日 22:00
- 实时同步：可选

## 🐛 常见问题

### Q: 飞书授权失败？
**A**: 在飞书中重新授权，确认所有权限已勾选。

### Q: 文档创建失败？
**A**: 检查网络连接，或改用本地模式。

### Q: 如何自定义分类？
**A**: 修改 `config.json` 中的 `categories` 字段。

### Q: 能导出到其他平台吗？
**A**: 支持导出 Markdown，可导入 Notion/Obsidian。
